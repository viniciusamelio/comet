# Comet React Native RPC Example

This Expo app consumes the typed TypeScript client generated by:

```sh
cargo run -p comet-cli -- rpc generate \
  --lang ts \
  --path examples/cloudflare-worker \
  --out examples/react-native-rpc/src/comet-rpc.ts
```

It talks to the task routes in `examples/cloudflare-worker`:

- `listTasks()`
- `createTask({ title })`
- `completeTask(id)`

## Run

Start the Worker API first:

```sh
cd examples/cloudflare-worker
npm install
npm run dev
```

Then start the React Native app:

```sh
cd examples/react-native-rpc
npm install
npm run start
```

In the app, set the API base URL and a bearer token. The generated RPC client
adds `Authorization: Bearer <token>` for routes marked as authenticated in the
manifest.

For local devices, use a URL reachable from the device or simulator. For
example, Android emulators commonly need `http://10.0.2.2:<port>` instead of
`localhost`.
